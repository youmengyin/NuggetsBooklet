####  11.4 章 贡献指南

**为什么贡献 OpenClaw 值得你投入时间？**

-   项目星标已超 280k，增长极快（每天 1k+）
-   创始人非常活跃，PR 回复率 > 90%，合并速度快
-   贡献者能优先内测 alpha 功能、获得 ClawHub 官方标签、甚至被邀请加入核心团队
-   最重要的是：你改的代码，会被成千上万的人每天使用

#####  完整 PR 流程（复制即用）

1.  **准备开发环境** （5 分钟）
1.  Bash

```
git clone https://github.com/openclaw/openclaw.gitcd openclawpnpm installpnpm dev:gateway   # 先跑起来验证环境
```

1.  **创建分支** （规范命名）
1.  Bash

```
git checkout -b feat/add-dingding-channel    # 新功能# 或git checkout -b fix/browser-sandbox-leak     # 修复 Bug# 或git checkout -b docs/zh/improve-install-guide # 文档
```

1.  **开发 & 测试**

    1.  修改代码 / 文档
    1.  运行测试：pnpm test
    1.  自测：本地跑 Gateway + 新 Channel/Node
    1.  提交前：pnpm lint && pnpm format

1.  **提交 Commit** （推荐格式）

1.  text

```
feat(channel-dingtalk): add initial support for DingTalk group bot- implement message handler- add OAuth flow- write basic SKILL.md exampleCloses #1234
```

1.  **推送 & 创建 PR**
1.  Bash

```
git push origin feat/add-dingding-channel
```

1.  然后去 GitHub 创建 Pull Request：

    1.  标题：保持和 Commit 一致

    1.  描述：写清楚

        -   做了什么
        -   为什么需要
        -   如何测试
        -   截图 / 日志（强烈推荐）

    1.  打上标签（feat / fix / docs / security）

1.  **等待 Review**

    1.  steipete 或 Maintainer 通常 1–3 天内回复
    1.  有修改意见 → 本地改 → force push
    1.  绿灯后会合并

#### 11.5 AI 辅助编码规范（社区最流行玩法）

让你的 Agent 帮你写代码！（第12章自迭代进阶版）

**在 Dashboard 输入模板** ：

text

```
请帮我写一个新的 Channel 适配器：支持飞书群聊。要求：- 基于 @openclaw 触发- 支持富文本、卡片消息、文件上传- 完整 packages/channel-feishu/ 目录结构- 包含 auth、message normalize、sendMessage 实现- 写好 README + 测试用例- 符合 OpenClaw 编码规范（eslint + 类型安全）
```

Agent 会直接输出完整代码文件夹内容，你复制到本地 → commit → PR。

**推荐 Prompt 优化** （加到你的 SOUL.md）：

text

```
你现在是 OpenClaw 核心开发者。输出代码必须：1. 符合现有 packages/gateway/src/ws/protocol.ts 协议2. 使用 TypeScript + 严格类型3. 添加 JSDoc 注释4. 包含单元测试路径5. 安全第一：所有外部输入校验 + 沙箱考虑
```
