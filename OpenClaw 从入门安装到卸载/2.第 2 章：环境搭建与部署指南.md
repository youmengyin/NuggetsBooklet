# 第 2 章：环境搭建与部署指南

## 前言

读完第 1 章后，你应该对 OpenClaw 的能力有了基本认识。这一章将带你完成从环境检查到看见 Web 控制台的完整流程。

无论你用的是 Mac、Windows 还是 Linux，无论你想装在自己电脑上还是云服务器上，跟着本章走，最快 5 分钟就能完成部署。

## 学习目标

读完本章，你将掌握以下核心技能：

1. **环境检查与 Node.js 安装**：学会安装和验证 Node.js 版本，确保运行环境满足 OpenClaw 的最低要求
2. **多平台部署能力**：根据自身操作系统（Mac / Windows / Linux）选择最优安装路线，完成 OpenClaw 的本地部署
3. **Docker 容器化部署**：掌握使用 Docker Compose 一键部署 OpenClaw 的完整流程，适用于云端 7x24 小时运行场景
4. **Onboarding 向导配置**：完成首次启动的交互式配置，包括 LLM API Key 绑定、技能安装和 Web 控制台访问
5. **常见问题排查**：具备独立解决安装过程中 5 种以上常见报错的能力

## 2.1 前置准备：环境检查清单

在动手安装之前，先花 2 分钟检查一下你的电脑是否满足以下条件：

| 检查项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| **Node.js** 版本 | >= v22.16 | Node 24 LTS 最新版 |
| 内存（RAM） | 4 GB | 8 GB 及以上 |
| 磁盘空间 | 2 GB 可用 | 5 GB 及以上 |
| 网络环境 | 能访问 npm 仓库 | 稳定的国际网络或配置镜像源 |
| 操作系统 | macOS 12+ / Windows 10+ / Ubuntu 20.04+ | macOS 最新版 |

其中，**Node.js 是唯一的硬性依赖**。如果你的电脑上还没有安装 Node.js，或者版本低于 v22.16，下面会教你从零开始安装。

### 2.1.1 检查 Node.js 是否已安装

打开终端（macOS / Linux）或 PowerShell（Windows），输入：

```bash
node -v
```

如果已经安装且版本满足要求，你会看到类似这样的输出：

```
v24.x.x
```

如果提示 `command not found`（macOS / Linux）或 `'node' 不是内部或外部命令`（Windows），说明你还没有安装 Node.js，请继续往下看。

如果版本号低于 `v22.16`（比如显示 `v18.x.x` 或 `v20.x.x`），也需要升级，直接跟着下面的步骤操作即可。

### 2.1.2 手动安装 Node.js（不用 nvm 的保守方案）

如果官方一键安装脚本因为网络、权限或公司安全策略无法使用，你可以先手动安装 Node.js，再通过 npm 安装 OpenClaw。当前推荐使用 **Node 24 LTS**；如果你的团队仍统一在 Node 22，也请确保版本至少为 **22.16+**。

> **补充说明**：如果你平时习惯用版本管理工具，也可以使用 `nvm`、`vfox`、`volta` 等来安装 Node.js；只是本章的手动安装步骤统一按 **Node.js 官网安装包 / 官方二进制包** 来写，减少初次上手时的分支复杂度。

#### macOS：下载 `.pkg` 安装包

1. 打开 [Node.js 官网下载页](https://nodejs.org/en/download)，选择 **Node 24 LTS**。
2. 下载适用于 macOS 的 `.pkg` 安装包，双击后按向导一路安装。
3. 安装完成后，重新打开终端，执行：

```bash
node -v
npm -v
```

如果 `node -v` 输出为 `v24.x.x`，或至少 `v22.16+`，说明安装完成。

#### Windows：下载 `.msi` 安装包

1. 打开 [Node.js 官网下载页](https://nodejs.org/en/download)，选择 **Node 24 LTS**。
2. 下载适用于 Windows 的 `.msi` 安装包，双击安装，保留默认配置即可。
3. 安装完成后，关闭并重新打开 PowerShell，执行：

```powershell
node -v
npm -v
```

如果命令能正常输出版本号，说明 Node.js 已经加入系统环境变量。

#### Linux：下载官网二进制包

1. 打开 [Node.js 官网下载页](https://nodejs.org/en/download) 或对应版本归档页，选择与你机器架构匹配的 Linux 二进制包（常见为 `linux-x64` 或 `linux-arm64` 的 `.tar.xz` 文件）。
2. 将压缩包解压到你希望长期保存的位置，例如 `/usr/local/lib/nodejs/`。
3. 把解压目录下的 `bin` 路径加入当前 shell 的 `PATH`，然后重新加载 shell 配置。
4. 执行下面的命令验证：

```bash
node -v
npm -v
```

如果你只是想尽快装好 OpenClaw，而不想自己处理 Linux 二进制包和 `PATH`，建议优先使用下一节的官方一键脚本。

#### 手动安装 OpenClaw

Node.js 准备完成后，再执行：

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

安装完成后，你还可以执行下面两条命令做快速检查：

```bash
openclaw doctor
openclaw dashboard
```

## 2.2 主流环境部署路线图

根据你的操作系统和使用场景，建议优先选择官方安装脚本；只有脚本不可用时，再退回到 2.1.2 节的手动安装方案。

| 部署方式 | 适用场景 | 难度 | 推荐指数 |
|----------|----------|------|----------|
| macOS 本地一键安装 | Mac 日常使用、桌面控制 | 简单 | ★★★★★ |
| Windows 安装（原生 / WSL2） | Windows 用户 | 中等 | ★★★★☆ |
| Linux 一键安装 | Linux 桌面、VPS、云主机 | 简单 | ★★★★★ |
| Docker 部署 | 7x24 小时无头运行 | 中等 | ★★★★★ |

### 2.2.1 路线 A：macOS 本地一键安装（最佳体验）

如果你使用 Mac，优先推荐官方一键安装脚本。根据官方安装文档，它会自动处理 Node 检测、安装 CLI，并启动新手引导。

打开终端，执行：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

如果你在安装过程中跳过了新手引导，后续手动执行：

```bash
openclaw onboard --install-daemon
```

如果脚本因为网络或权限问题无法使用，就回到 2.1.2 节先安装 Node.js，再手动执行：

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

### 2.2.2 路线 B：Windows 安装（原生 PowerShell / WSL2）

官方文档明确支持 Windows PowerShell 安装，但同时也**强烈建议在 WSL2 下运行 OpenClaw**。你可以按需求选择：

#### 方案一：Windows 原生 PowerShell 安装

如果你希望 OpenClaw 直接运行在当前 Windows 用户环境下，先打开 PowerShell，执行官方脚本：

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

如果脚本阶段跳过了引导，继续执行：

```powershell
openclaw onboard --install-daemon
```

如果你不想使用脚本，也可以先按 2.1.2 节从 Node.js 官网安装 `.msi`，再执行：

```powershell
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

> **避坑提示**：Windows 原生环境下，某些依赖本地编译工具链的包更容易报错；如果你主要做后台任务、开发或长期运行服务，优先考虑下面的 WSL2 方案。

#### 方案二：WSL2 安装（更适合后台任务 / 开发场景）

如果你更在意稳定性，或者想尽量贴近 Linux 服务器环境，先以管理员身份打开 PowerShell，执行：

```powershell
wsl --install
```

重启电脑并完成 Ubuntu 用户初始化后，在 Ubuntu 终端中执行 Linux 的官方脚本：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

如果你在 WSL2 里跳过了引导，则执行：

```bash
openclaw onboard --install-daemon
```

> **重要提醒**：如果你需要 OpenClaw 直接控制宿主机 Windows 桌面，应该优先选择上面的原生 PowerShell 安装；WSL2 更适合后台执行和开发环境。

### 2.2.3 路线 C：Linux 一键安装（桌面 / VPS 都适用）

Linux 用户同样优先使用官方脚本，桌面环境、VPS、云主机都适用：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

如果你跳过了新手引导，继续执行：

```bash
openclaw onboard --install-daemon
```

如果脚本不可用，再回到 2.1.2 节，先从 Node.js 官网下载 Linux 二进制包并完成 `PATH` 配置，然后执行：

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

### 2.2.4 路线 D：Docker 部署（适合 7x24 小时全天候运行）

如果你不希望自己的电脑 24 小时开机，把 OpenClaw 部署到云服务器是最佳选择。Docker 容器化部署让整个过程标准化、可复现，也便于后续升级维护。

#### 云服务器选型建议

在选择云服务器之前，先看看主流云厂商的对比：

| 对比项 | 腾讯云轻量应用服务器 | 阿里云 ECS | AWS EC2 |
|--------|---------------------|------------|---------|
| 推荐配置 | 2 核 4G | 2 核 4G | t3.medium |
| 月费参考 | 约 50-80 元 | 约 60-100 元 | 约 $30-40 |
| 国内访问速度 | 快 | 快 | 需要中转 |
| 适合人群 | 国内个人用户首选 | 企业用户 / 有阿里云生态 | 海外用户 / 需要海外 API 直连 |
| 操作系统推荐 | Ubuntu 22.04 | Ubuntu 22.04 | Ubuntu 22.04 |
| 备案要求 | 需要（绑定域名时） | 需要（绑定域名时） | 不需要 |

> **Tips**：如果你主要使用海外大模型 API（如 OpenAI、Claude），选择海外服务器（如 AWS 东京 / 新加坡区域）可以获得更低的 API 延迟。如果主要使用国产大模型（如 DeepSeek、Kimi），国内服务器即可。

#### 安装 Docker 和 Docker Compose

SSH 连接到你的云服务器后，执行以下命令安装 Docker：

```bash
# 使用官方脚本一键安装 Docker
curl -fsSL https://get.docker.com | bash

# 将当前用户添加到 docker 组（免 sudo）
sudo usermod -aG docker $USER

# 重新登录使权限生效
exit
# 重新 SSH 连接后验证
docker --version
docker compose version
```

预期输出：

```
Docker version 27.x.x, build xxxxxxx
Docker Compose version v2.x.x
```

#### Docker Compose 完整配置

在服务器上创建项目目录并编写配置文件：

```bash
mkdir -p ~/openclaw && cd ~/openclaw
```

创建 `docker-compose.yml` 文件。下面这份是**案例化部署示例**，目的是让你快速跑起来；它不是 OpenClaw 官方唯一标准模板：

```yaml
services:
  openclaw:
    image: ghcr.io/openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "18789:18789"
    volumes:
      - ./workspace:/app/workspace
      - ./openclaw.json:/app/openclaw.json
    environment:
      - TZ=Asia/Shanghai
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
```

#### 配置文件

在同目录下创建 `openclaw.json` 配置文件。下面这份同样是示例，字段结构按当前官方配置口径写，但具体模型、挂载路径和认证策略要按你的环境调整：

```json
{
  "models": {
    "providers": {
      "openai": {
        "apiKey": "sk-your-api-key-here"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "openai/gpt-5"
      }
    }
  },
  "gateway": {
    "port": 18789,
    "bind": "lan",
    "auth": {
      "mode": "token",
      "token": "replace-with-a-long-random-token"
    }
  },
  "logging": {
    "level": "info"
  }
}
```

如果使用国产大模型（如 DeepSeek），就在 `models.providers` 中添加对应提供商配置，并把 `agents.defaults.model.primary` 改成对应的 `provider/model`。

> **安全提醒**：`openclaw.json` 中包含你的 API Key 等敏感信息，请确保文件权限设置正确：
> ```bash
> chmod 600 ~/openclaw/openclaw.json
> ```

#### 启动服务

```bash
cd ~/openclaw
docker compose up -d
```

启动成功后，你会看到容器 `Started` 的提示。

查看运行状态：

```bash
docker compose ps
```

确认 `STATUS` 列显示 `Up` 且端口映射包含 `18789` 即表示运行正常。

查看实时日志：

```bash
docker compose logs -f openclaw
```

## 2.4 新手避坑指南（Troubleshooting）

安装过程中遇到问题是正常的，下面列出了最常见的 8 个问题及解决方案。

### 问题 1：`openclaw: command not found`

**原因**：npm 全局安装目录没有被添加到系统的 `PATH` 环境变量中。

**解决方案**：

首先，找到 npm 全局安装路径：

```bash
npm config get prefix
```

预期输出类似：

```
/usr/local
```

然后将 `bin` 子目录添加到 `PATH`：

```bash
# macOS / Linux（zsh）
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# macOS / Linux（bash）
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

Windows 用户需要在"系统属性 -> 环境变量 -> Path"中手动添加 npm 全局路径（通常是 `C:\Users\你的用户名\AppData\Roaming\npm`）。

### 问题 2：图像库 `sharp` 安装报错

**原因**：`sharp` 是一个依赖本地 C++ 编译的图像处理库，在某些环境下预编译二进制文件下载失败。

**解决方案**：

```bash
# macOS：安装 sharp 的系统依赖
brew install vips

# Ubuntu / Debian：安装 sharp 的系统依赖
sudo apt-get install -y libvips-dev

# 安装依赖后重新安装 OpenClaw
npm install -g openclaw@latest
```

### 问题 3：`EACCES: permission denied`

**原因**：npm 全局安装时权限不足。

**解决方案**：

**不要使用 `sudo npm install -g`**，这会导致后续更多权限问题。优先做法是使用官方安装脚本，或按 2.1.2 节先用 Node.js 官网安装包准备好 Node 环境，再执行全局安装。

如果你已经用 sudo 安装过，可以先修复当前 npm 全局目录权限：

```bash
sudo chown -R $(whoami) $(npm config get prefix)/{lib/node_modules,bin,share}
```

如果你希望彻底避免系统级目录权限问题，也可以把 npm 全局安装目录改到当前用户目录：

```bash
mkdir -p ~/.npm-global
npm config set prefix ~/.npm-global
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 问题 4：`ETIMEOUT` 或 `ECONNREFUSED`（网络超时）

**原因**：国内网络访问 npm 官方仓库速度慢或被阻断。

**解决方案**：

```bash
# 切换到国内镜像源
npm config set registry https://registry.npmmirror.com

# 验证镜像源是否生效
npm config get registry

# 然后重新安装
npm install -g openclaw@latest
```

### 问题 5：Docker 容器启动后立即退出

**原因**：通常是环境变量配置错误或端口冲突。

**解决方案**：

```bash
# 查看容器退出日志
docker compose logs openclaw

# 常见原因 1：端口被占用
# 检查 18789 端口是否被其他程序占用
lsof -i :18789

# 如果端口被占用，修改 docker-compose.yml 中的端口映射
# 例如将宿主机端口改为 28789：
# ports:
#   - "28789:18789"

# 常见原因 2：配置文件格式错误
# 确保 openclaw.json 是合法的 JSON5 格式
```

### 问题 6：Onboarding 向导中 LLM 连接验证失败

### 问题 8：Node.js 安装完成后，`node -v` 仍然报错

**原因**：新安装的 Node.js 路径还没有被当前终端会话加载，或者 Linux 手动安装时没有把解压目录的 `bin` 加入 `PATH`。

**解决方案**：

```bash
# macOS / Windows：先完全关闭当前终端，再重新打开后重试
node -v

# Linux 手动安装包场景：确认 PATH 已包含 Node 的 bin 目录
echo 'export PATH="/usr/local/lib/nodejs/<node-dir>/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
node -v
```

如果你使用的是 zsh，把上面的 `~/.bashrc` 改成 `~/.zshrc` 即可。
