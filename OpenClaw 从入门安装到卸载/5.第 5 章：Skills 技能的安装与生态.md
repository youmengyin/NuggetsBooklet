# 第 5 章：Skills 技能的安装与生态

> 在上一章，我们已经把 OpenClaw 接到了飞书等即时通讯工具里。但只会接消息、回消息还不够。很多具体任务还要靠额外能力来完成，比如搜索网页、处理文件、调用外部服务。在 OpenClaw 里，这类扩展通常通过 **Skills（技能）** 提供。本章会讲清楚 Skill 是什么、怎么装、怎么管，以及怎么避免装到有风险的内容。

## 本章学习目标

*   理解 Skills 的技术架构与 SKILL.md 文件规范
*   掌握 ClawHub 技能市场的搜索与安装流程
*   了解自定义 Skill 的开发方式
*   建立 Skills 安全风控意识与实操能力
*   掌握技能的更新与同步策略

### 1. 什么是 Skills？先把三个概念分清

这一章最容易混淆的，是 **Tool**、**Skill** 和 **ClawHub** 其实不是一回事：

*   **Tool（工具）**：Agent 在执行任务时可以直接调用的能力，比如浏览器操作、文件读写、Shell 命令、记忆检索。
*   **Skill（技能）**：把说明文档、提示词、配置文件等内容打包在一起的能力包，核心文件通常是 `SKILL.md`。
*   **ClawHub**：分发和管理 Skill 的市场与命令行入口，站点是 `clawhub.ai`。

可以把它理解成：Tool 是“能做什么”，Skill 是“怎么把这件事组织好并交给 Agent 用”，ClawHub 是“去哪里找和安装这些 Skill”。

无论是查天气、抓网页、操作 Notion 数据库，还是读取 GitHub 仓库，都可以先去 ClawHub 看看有没有现成的 Skill。

#### 1.1 Skill 的技术架构

每个 Skill 以一个 `SKILL.md` 文件为核心。这个文件使用 YAML 前置元数据（frontmatter）来声明技能的名称、描述和工具定义，正文部分则是给 AI 智能体的详细使用说明。理解这个结构，对你后续安装、调试甚至自己开发 Skill 都至关重要。

一个标准的 `SKILL.md` 文件示例如下：

```markdown
---
name: my-custom-skill
description: A custom skill description
tools:
  - name: get_weather
    description: 获取指定城市的当前天气信息
    parameters:
      type: object
      properties:
        city:
          type: string
          description: 城市名称，如：北京、上海
---

# My Custom Skill

这里是给智能体的详细使用说明。当用户请求天气信息时，
使用 get_weather 工具查询，并以简洁的格式返回结果。
```

几个关键要素说明：

*   **YAML frontmatter**：`---` 之间的部分是技能的元数据，包括名称、描述和工具定义。OpenClaw 通过解析这部分来识别和加载技能。
*   **tools**：在 frontmatter 中声明该技能提供的工具列表，每个工具包含名称、描述和参数定义。
*   **正文（Markdown body）**：`---` 之后的部分是给 AI 智能体的指令和上下文说明，会在调用该 Skill 时自动注入到 AI 的提示词中。

**技能的三个加载目录：**

OpenClaw 从以下三个目录加载技能，优先级从高到低：

| 目录类型          | 路径                          | 说明                        |
| ------------- | --------------------------- | ------------------------- |
| **workspace** | 项目根目录下的 `.openclaw/skills/` | 工作区级别的自定义技能，仅在当前项目中生效     |
| **managed**   | `~/.openclaw/skills/`       | 通过 clawhub 安装和管理的技能       |
| **bundled**   | OpenClaw 安装目录内置             | 随 OpenClaw 一起分发的内置技能，开箱即用 |

当多个目录中存在同名技能时，workspace 级别的优先级最高，可以覆盖 managed 和 bundled 中的同名技能。

### 2. ClawHub 技能市场：搜索、筛选与安装全流程

ClawHub 是 OpenClaw 官方的技能分发平台，类似于 VS Code 的扩展市场或 npm 的包管理器。你可以通过 Web 界面或命令行两种方式来使用它。

#### 2.1 通过 Web 界面浏览

打开浏览器访问 `https://clawhub.ai`，你会看到一个类似应用商店的界面。首页展示了热门推荐、最新上架和分类导航。


![image.png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd3beb14bc6b4346a0342faf8f897af0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2584&h=2488&s=1075332&e=png&b=1b1510)

每个技能卡片上会显示以下关键信息：

*   **名称与描述**：技能的功能简介。
*   **作者与来源标签**：`Official`（官方）、`Verified`（已验证）、`Community`（社区）。
*   **下载量与评分**：社区使用热度的直观指标。
*   **兼容版本**：该技能支持的 OpenClaw 版本范围。

#### 2.2 通过命令行操作（推荐）

对于开发者来说，命令行是更高效的方式。安装、更新、同步主要通过 `clawhub` 完成；`openclaw skills` 主要用于列出、查看和检查本地技能状态。

**安装 Clawhub**

```bash
npm install -g clawhub@latest
```
安装以后可以使用 `clawhub` 命令

**搜索技能：**

```bash
# 按关键词搜索
clawhub search browser

# 搜索特定领域的技能
clawhub search "file manager"
```

**安装技能：**

```bash
# 安装指定技能
clawhub install browser
```

**查看已安装技能：**

```bash
# 列出所有已安装的技能
openclaw skills list
```

安装完成后，OpenClaw 会自动将技能加载到运行时环境中，无需重启服务。

#### 2.3 Skills 分类速查表

为了帮你快速定位所需技能，这里按功能维度整理了一份分类速查表：

| 分类        | 代表技能                                          | 功能说明                      |
| --------- | --------------------------------------------- | ------------------------- |
| **网络与浏览** | `browser`（内置）、`web-scraper`、`api-caller`      | 网页交互、数据爬取、API 调用          |
| **文件与存储** | `file-manager`、`cloud-sync`、`pdf-reader`      | 本地文件操作、云存储同步、PDF 解析       |
| **通讯与协作** | `email-sender`、`slack-bot`、`feishu-ext`       | 邮件发送、Slack/飞书消息推送         |
| **数据与分析** | `data-analyzer`、`chart-generator`、`sql-query` | 数据分析、图表生成、数据库查询           |
| **开发与运维** | `git-operator`、`docker-manager`、`ci-trigger`  | Git 操作、Docker 管理、CI/CD 触发 |
| **多媒体**   | `image-gen`、`tts-engine`、`video-summarizer`   | 图片生成、语音合成、视频摘要            |

> 注意：`browser` 是 OpenClaw 的内置浏览器工具，不需要从 ClawHub 单独安装。上表中除 `browser` 外的技能名称来自社区生态，安装前请在 ClawHub 上确认其可用性和来源。

### 3. 常用技能介绍与配置

当你打开 Web 控制台的技能面板，或者通过命令行准备安装时，面对海量选择可能会无从下手。以下是一些常用技能的介绍和配置说明。

#### 3.1 browser：内置浏览器工具

`browser` 是 OpenClaw 的内置浏览器工具，无需从 ClawHub 单独安装。它赋予 AI 完整的网页交互能力——打开网页、点击按钮、填写表单、截取屏幕截图，甚至执行 JavaScript。底层基于 Playwright，支持 Chromium、Firefox 和 WebKit 三大浏览器引擎。

由于 `browser` 是内置工具，它随 OpenClaw 一起分发，开箱即用。如果需要自定义配置（如代理设置），可以参考 OpenClaw [官方文档](https://docs.openclaw.ai/tools/browser)中关于内置工具的配置说明。



**使用示例：**
安装 OpenClaw 后，你可以直接在对话中这样使用：

*   "帮我打开掘金首页，截图给我看看今天的热门文章"
*   "登录我的 GitHub，查看最新的 Pull Request 列表"
*   "打开这个网址，把页面上的表格数据提取成 CSV 文件"

#### 3.2 社区技能安装示例：subagent-driven-development

> 以下技能来自社区生态，安装前请在 ClawHub 上确认其最新状态和可用性。

`subagent-driven-development` 是一个社区贡献的技能，它让你的 AI 可以将复杂任务拆解成子任务，分发给多个子智能体并行处理，最后汇总审核结果。适合处理大型项目的代码审查、批量文档生成、多维度数据分析等场景。

**安装：**

```bash
clawhub install subagent-driven-development
```

#### 3.3 技能的更新与同步

通过 `clawhub` 命令行工具可以方便地管理已安装技能的更新：

```bash
# 更新所有已安装的 managed 技能
clawhub update

# 同步技能状态
clawhub sync
```

建议定期执行 `clawhub update` 以获取最新的安全修复和功能改进。

### 4. 自定义 Skill 开发入门

当 ClawHub 上找不到满足你需求的技能时，你完全可以自己动手开发一个。自定义 Skill 的核心就是编写一个 `SKILL.md` 文件。

#### 4.1 创建 SKILL.md

在你的工作区 `.openclaw/skills/` 目录下创建一个新文件夹，然后在其中创建 `SKILL.md` 文件：

```bash
mkdir -p .openclaw/skills/my-weather-skill
```

然后编写 `SKILL.md` 文件：

```markdown
---
name: my-weather-skill
description: 获取指定城市的当前天气信息
tools:
  - name: get_weather
    description: 查询城市天气
    parameters:
      type: object
      properties:
        city:
          type: string
          description: 城市名称，如：北京、上海
      required:
        - city
---

# Weather Skill

当用户询问天气信息时，使用 get_weather 工具查询指定城市的天气。
返回结果应包含温度、天气状况和湿度信息，以简洁友好的格式呈现。

注意事项：
- 如果用户没有指定城市，请先询问。
- 支持中文和英文城市名。
```

#### 4.2 工作区技能的加载

将 `SKILL.md` 放在工作区的 `.openclaw/skills/my-weather-skill/` 目录下后，OpenClaw 会自动识别并加载该技能。工作区级别的技能优先级最高，可以覆盖同名的 managed 或 bundled 技能。

你可以通过以下命令确认技能是否被正确加载：

```bash
openclaw skills list
```

#### 4.3 开发建议

*   在 YAML frontmatter 中清晰定义工具的参数类型和描述，这直接影响 AI 调用工具的准确性。
*   Markdown 正文部分是给 AI 的"使用手册"，写得越清晰，AI 的表现越好。
*   先在工作区目录中开发和测试，确认稳定后再考虑提交到 ClawHub 社区。

### 5. Skills 安全风控

**这一节非常重要。** 第三方 Skill 让 OpenClaw 更强，但也会带来供应链风险。最稳妥的做法是把每个 Skill 都当成“需要审查的外部代码”，先看来源、再看权限、最后再决定是否安装。

#### 5.1 安全评估原则

在安装任何第三方技能之前，建议遵循以下评估流程：

1.  **检查权限声明**：仔细阅读技能的 `SKILL.md` 文件，查看它声明了哪些工具和参数。一个"天气查询"技能不应该需要文件写入能力。
2.  **审查来源**：优先选择官方认证（`Official`）或已验证（`Verified`）来源的技能。在 ClawHub 网页上可以看到来源标签。
3.  **阅读代码**：对于社区技能，建议在安装前阅读其源代码，检查是否存在混淆代码、Base64 编码的可疑载荷、或未声明的外部网络请求。

#### 5.2 安全最佳实践清单

1.  **隔离环境测试**：如果遇到来源不明但又想尝试的社区新技能，强烈建议在 Docker 容器或虚拟机（VM）等隔离环境中先行测试，不要直接在物理宿主机上盲目运行。

2.  **密钥管理（防泄漏）**：在给 Agent 编写配置时，**绝对不要**将 API Key 等高权限敏感数据直接硬编码写在提示词或明文配置里，应统一使用环境变量来管理。
    ```bash
    # 正确做法：使用环境变量
    export WEATHER_API_KEY="your-api-key-here"

    # 错误做法：硬编码在配置文件中（千万不要这样做！）
    # { "api_key": "sk-xxxxxxxxxxxx" }
    ```

3.  **认准官方标志**：尽量只安装官方认证（`Official`）或已验证（`Verified`）来源的 Skills。

4.  **权限最小化原则**：安装技能时仔细审查其工具定义中的参数和能力范围，拒绝不合理的权限申请。

5.  **定期更新**：通过 `clawhub update` 保持已安装技能为最新版本，及时获取安全修复。

### 6. 技能更新与同步

随着你安装的技能越来越多，保持它们的更新很重要。OpenClaw 通过 `clawhub` 提供了简洁的更新机制：

```bash
# 更新所有已安装的 managed 技能到最新版本
clawhub update

# 同步技能状态
clawhub sync

# 查看当前已安装的技能
openclaw skills list
```

建议养成定期执行 `clawhub update` 的习惯，以获取最新的功能改进和安全修复。
