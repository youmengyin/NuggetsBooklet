# 第 4 章：接入即时通讯工具（IM），打造移动办公入口

> 在上一章，我们已经把 OpenClaw 的模型接通了。但如果你每次还要专门打开网页控制台和它对话，使用成本还是很高。本章会把它接到你平时就在用的即时通讯软件（IM）里。接好以后，你可以直接在飞书、Telegram 等聊天窗口里给它发消息，让它在熟悉的工作流里处理任务。

## 本章学习目标

- 理解 AI 接入 IM 的核心价值与架构原理
- 掌握飞书 Channel 的完整接入、配置与调优流程
- 学会创建 Telegram Bot 并完成 OpenClaw 对接
- 了解 WhatsApp、Discord 等平台的接入方式
- 熟悉群聊管理的配置策略（私聊策略、群聊策略、@触发等）
- 能够根据业务场景选择最合适的 IM 平台

---

### 1. 为什么把 AI 接入 IM 更省事？

传统的网页端 AI 往往是"信息孤岛"，你需要手动复制聊天记录和文档内容。而把 OpenClaw 接入 IM（如飞书、Telegram、WhatsApp）后，最大的变化是下面这三点：

1. **原生上下文读取**：AI 可以直接读取群聊历史、查看群文件、获取话题（Thread）上下文。你不再需要手动复制粘贴，AI 天然就在信息流中，能感知完整的对话脉络。
2. **移动办公更顺手**：在通勤路上，你可以直接通过手机飞书发一段语音，让 AI 在后台继续处理文档整理、状态查询这类任务。
3. **主动通知与触达**：配合第 3 章提到的定时任务（Cron），AI 可以在每天早上 9 点主动往你的工作群发送早报，在项目截止日前自动提醒团队成员，真正实现从"被动应答"到"主动服务"的跃迁。

从技术架构上看，OpenClaw 的 IM 接入采用了**插件化的 Channel 架构**。每个 IM 平台对应一个独立的 Channel 插件，它们通过统一的消息协议与核心 Agent 引擎通信。这意味着你可以同时接入多个平台，而 AI 的核心逻辑和记忆是共享的——飞书群里讨论的项目进度，你在 Telegram 私聊中也能无缝追问。

### 2. 国内首选：飞书 Channel 接入实战

对于国内开发者和团队，强烈推荐首选**飞书**。相比于企业微信和钉钉，飞书开放平台的 API 更加现代化，对 Markdown 消息卡片、交互式按钮组件的支持也更为完善。OpenClaw 通过内置的 Channel 机制原生支持飞书接入，配置流程简洁高效。

#### 2.1 版本检查（前置条件）

在开始之前，建议先升级到较新的 2026.3 版本，再开始接入飞书，避免踩到旧版本兼容性问题：

| 操作系统 | 最低版本要求 | 检查命令 |
|---------|------------|---------|
| Linux / macOS | 建议 `2026.3.x` 及以上 | `openclaw -v` |
| Windows (WSL2) | 建议 `2026.3.x` 及以上 | `openclaw -v` |

如果版本过低，请运行以下命令升级：

```bash
npm install -g openclaw
```

#### 2.2 通过 CLI 添加飞书 Channel

打开终端，执行以下命令启动飞书 Channel 接入向导：

```bash
openclaw channels add
```

在交互式向导中选择 `feishu`，然后按提示输入你的飞书应用凭证（App ID 和 App Secret）。向导会自动将配置写入 `~/.openclaw/openclaw.json` 并完成连接测试。

#### 2.3 手动创建飞书应用（不使用 CLI 向导的方式）

如果你的企业网络环境无法执行上述 CLI 向导，或者你希望对飞书应用有更精细的控制权，也可以选择手动在飞书开放平台创建应用：

1. **登录飞书开放平台**：访问 `open.feishu.cn`，使用管理员账号登录。
2. **创建企业自建应用**：点击"创建应用" -> 选择"企业自建应用" -> 填写应用名称（如"OpenClaw 助手"）和描述。
3. **获取应用凭证**：在应用详情页的"凭证与基础信息"中，记录下 `App ID` 和 `App Secret`。
4. **配置机器人能力**：在"应用能力"中开启"机器人"功能，设置机器人名称和头像。
5. **配置事件订阅**：在"事件订阅"中订阅以下事件：
   - `im.message.receive_v1`（接收消息）
   - `im.message.reaction.created_v1`（消息表情回复）
   - `im.chat.member.bot.added_v1`（机器人进群）
6. **发布应用**：提交审核并发布（企业内部应用通常可以自动通过）。
7. **写入 OpenClaw 配置**：将获取的凭证手动写入 `openclaw.json`：

```json
{
  "channels": {
    "feishu": {
      "accounts": {
        "my-bot": {
          "appId": "cli_xxxxxxxxxx",
          "appSecret": "xxxxxxxxxxxxxxxxxxxxxxxx"
        }
      },
      "connectionMode": "websocket",
      "dmPolicy": "pairing",
      "groupPolicy": "allowlist",
      "groups": {
        "oc_你的群ID": {
          "requireMention": true
        }
      }
    }
  }
}
```

**配置项说明：**

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `accounts.<name>.appId` | string | 飞书应用的 App ID |
| `accounts.<name>.appSecret` | string | 飞书应用的 App Secret |
| `connectionMode` | string | 连接模式，推荐 `"websocket"`，也可选 `"webhook"` |
| `dmPolicy` | string | 私聊策略，常见值有 `"pairing"`、`"allowlist"`、`"open"`、`"disabled"` |
| `groupPolicy` | string | 群聊策略，常见值有 `"open"`、`"allowlist"`、`"disabled"` |
| `channels.feishu.groups.<chat_id>.requireMention` | boolean | 针对特定群组是否要求 @ 触发 |

#### 2.4 飞书权限配置详解

无论你使用 CLI 向导还是手动创建，飞书机器人要正常工作，都需要申请以下权限。在飞书开放平台的"权限管理"页面中，确保以下权限已开通：

**消息相关权限（必选）：**

| 权限标识 | 权限名称 | 用途说明 |
|---------|---------|---------|
| `im:message` | 获取与发送单聊、群组消息 | AI 收发消息的基础能力 |
| `im:message:send_as_bot` | 以应用身份发送消息 | 机器人主动发送消息 |
| `im:resource` | 获取消息中的资源文件 | 读取用户发送的图片、文件 |
| `im:chat` | 获取群组信息 | 读取群名称、群成员列表 |
| `im:chat:readonly` | 获取群组信息（只读） | 查看群聊基本信息 |

**飞书办公套件权限（可选，需在飞书开放平台手动申请）：**

| 权限标识 | 权限名称 | 用途说明 |
|---------|---------|---------|
| `docs:doc` | 查看、创建和编辑云文档 | AI 帮你写飞书文档 |
| `sheets:spreadsheet` | 查看、创建和编辑电子表格 | AI 操作多维表格 |
| `calendar:calendar` | 查看和管理日历 | AI 帮你排日程 |
| `drive:drive` | 查看和管理云空间文件 | AI 读取云盘文件 |
| `bitable:bitable` | 查看和管理多维表格 | AI 操作多维表格数据 |

> 权限申请后需要管理员审批。如果你本身就是企业管理员，可以直接在"权限管理"页面点击"批量开通"。

#### 2.5 验证连接

配置完成后，在飞书中找到这个机器人并发送任意消息，验证 AI 是否正常响应。

1. **验证连接**：在飞书中打开与机器人的单聊窗口，发送一条普通消息（如"你好"）。如果机器人正常回复，说明 Channel 连接成功。


![image.png](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5b288456724b40ae8bb22a05095b77ed~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1126&h=500&s=74261&e=png&b=fdfcfc)

2. **检查日志**：如果机器人没有响应，可以在终端查看 OpenClaw 的运行日志，排查连接或权限问题。

### 3. 进阶调优：连接模式与群聊策略配置

飞书 Channel 的默认配置偏向保守，你可以根据实际场景灵活调整。所有配置都在 `openclaw.json` 的 `channels.feishu` 节点下。

#### 3.1 连接模式选择

OpenClaw 支持两种连接模式：

```bash
# WebSocket 模式（推荐，无需公网 IP）
openclaw config set channels.feishu.connectionMode websocket

# Webhook 模式（适合已有公网服务的场景）
openclaw config set channels.feishu.connectionMode webhook
```

**参数说明：**

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| `channels.feishu.connectionMode` | string | `"websocket"` | 连接模式，`websocket` 或 `webhook` |

WebSocket 模式不需要公网 IP，适合本地开发和内网部署；Webhook 模式适合已有公网服务器的生产环境。

如果你选择 `webhook` 模式，还要补齐飞书要求的 `verificationToken` 和 `encryptKey`，否则事件订阅验证不会通过。

#### 3.2 私聊策略（dmPolicy）

控制机器人是否响应用户的私聊消息：

```bash
# 私聊先走配对流程（推荐默认值）
openclaw config set channels.feishu.dmPolicy pairing

# 对所有私聊开放
openclaw config set channels.feishu.dmPolicy open
```

#### 3.3 群聊策略（groupPolicy）

控制机器人在群聊中的响应行为：

```bash
# 只处理已列入 allowlist 的群
openclaw config set channels.feishu.groupPolicy allowlist

# 对群内消息开放
openclaw config set channels.feishu.groupPolicy open
```

**针对特定群组的独立策略：**

如果你想在某个特定群里覆盖全局策略：

```bash
# 获取群组 ID（在飞书群设置中可以找到）
# 然后为该群单独配置是否需要 @
openclaw config set channels.feishu.groups.oc_你的群ID.requireMention true --json
```

#### 3.4 多账号管理

OpenClaw 支持在同一个飞书 Channel 下配置多个机器人账号，适合需要区分不同业务场景的团队：

```json
{
  "channels": {
    "feishu": {
      "accounts": {
        "customer-service": {
          "appId": "cli_aaa",
          "appSecret": "secret_aaa"
        },
        "dev-assistant": {
          "appId": "cli_bbb",
          "appSecret": "secret_bbb"
        }
      },
      "connectionMode": "websocket",
      "dmPolicy": "pairing",
      "groupPolicy": "allowlist"
    }
  }
}
```

每个账号对应一个独立的飞书机器人应用，共享相同的策略配置。

### 4. 海外极客首选：Telegram Bot 完整接入指南

如果你的业务出海，或者个人更习惯使用海外通讯软件，Telegram 是最推荐的选择。它的开放生态极佳，对 Markdown 和按钮组件的支持最为完美，且完全免费、无需企业认证。

#### 4.1 创建 Telegram Bot（从 @BotFather 开始）

Telegram 的 Bot 创建流程非常简洁，全程在 Telegram 客户端内完成：

1. **打开 BotFather**：在 Telegram 搜索栏中搜索 `@BotFather`，点击进入对话。


![image.png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9480ee1d054d47df8d62bcd6a7e6a7d2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1016&h=1264&s=208333&e=png&b=050505)

2. **创建新 Bot**：发送 `/newbot` 命令，BotFather 会引导你完成创建：
   - 输入 Bot 的**显示名称**（如 `OpenClaw Assistant`）
   - 输入 Bot 的**用户名**（必须以 `bot` 结尾，如 `my_openclaw_bot`）

3. **获取 Bot Token**：创建成功后，BotFather 会返回一个形如 `123456789:ABCdefGHIjklMNOpqrsTUVwxyz` 的 Token。**请妥善保管这个 Token，它是你 Bot 的唯一凭证。**

![image.png](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/90a5470430814c439fcf59c10671dea1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1010&h=2184&s=1789491&e=png&b=2d232f)

4. **配置 Bot 信息（可选但推荐）**：
   ```
   /setdescription - 设置 Bot 描述（用户首次打开时看到）
   /setabouttext   - 设置 Bot 简介
   /setuserpic     - 设置 Bot 头像
   /setcommands    - 设置 Bot 命令菜单
   ```

   推荐设置以下命令菜单：
   ```
   start - 开始对话
   help - 获取帮助
   reset - 重置对话上下文
   status - 查看 Bot 状态
   ```

#### 4.2 将 Telegram Bot 接入 OpenClaw

获取 Token 后，有两种方式完成接入：

**方式一：通过 onboard 向导（推荐新手）**

```bash
openclaw onboard
```

在向导的 Channel 配置环节，选择 `Telegram`，然后粘贴你的 Bot Token 即可。默认情况下，Telegram 直接使用 Long polling；Webhook 只是可选模式，不是必填前提。

**方式二：手动写入配置文件**

编辑 `~/.openclaw/openclaw.json`，添加 Telegram Channel 配置：

```json
{
  "channels": {
    "telegram": {
      "botToken": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz",
      "allowFrom": [],
      "dmPolicy": "pairing",
      "groupPolicy": "allowlist",
      "groups": {
        "*": {
          "requireMention": true
        }
      }
    }
  }
}
```

**配置项详解：**

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| `botToken` | string | 必填 | BotFather 提供的 Bot Token |
| `allowFrom` | array | `[]` | 允许私聊使用的用户白名单；为空时按 `dmPolicy` 处理 |
| `dmPolicy` | string | 依配置而定 | 私聊策略，常见值有 `pairing`、`allowlist`、`open`、`disabled` |
| `groupPolicy` | string | 依配置而定 | 群聊策略，常见值有 `allowlist`、`open`、`disabled` |
| `groups.*.requireMention` | boolean | `true` | 群里是否必须先 @ 机器人再触发 |
| `webhook` | object / string | 可选 | Webhook 模式配置；Long polling 默认可用，不必强制填写 |

#### 4.3 Telegram 使用技巧

Telegram 的开放生态提供了一些实用能力：

**Inline 模式（任意聊天中调用）：**

Telegram 原生支持 Inline 模式，你可以在任何聊天窗口中通过 `@你的bot名 问题` 的方式快速调用 AI，无需切换到 Bot 对话。

> 注意：Inline 模式需要先在 BotFather 中通过 `/setinline` 命令开启。此功能由 Telegram 平台提供，OpenClaw 会自动适配。

**Privacy Mode 设置：**

Telegram 的 Bot 默认开启 Privacy Mode，无法读取群聊中未 @ 自己的消息。如果你希望 Bot 能响应群内所有消息，需要在 BotFather 中通过 `/setprivacy` 命令将 Privacy Mode 设置为 `Disable`。

### 5. 更多平台：Discord 接入简述

OpenClaw 同样支持 Discord，适合游戏社区、开源项目和技术社群场景。

#### 5.1 快速接入步骤

1. 访问 Discord Developer Portal（`discord.com/developers/applications`），创建一个新的 Application。
2. 在 Bot 页面中创建 Bot 并获取 Token。
3. 在 OAuth2 页面生成邀请链接，将 Bot 邀请到你的 Discord 服务器。
4. 按当前官方 channels 文档配置 Discord。字段口径和早期文章里常见的 `allowedGuilds`、`allowedChannels` 已经不一样，当前更常见的是 `guilds`、`threadBindings` 这类配置。例如：

```json
{
  "channels": {
    "discord": {
      "groupPolicy": "allowlist",
      "guilds": {
        "你的服务器ID": {
          "requireMention": true
        }
      },
      "threadBindings": {
        "enabled": true
      }
    }
  }
}
```

Discord 的优势在于社区能力丰富，适合服务器、频道、线程都比较复杂的场景。

### 6. 选平台时怎么判断

这里不再给一张“绝对排名表”，因为不同团队的网络环境、审批流程和运营目标差异很大。更实用的判断方法是：

- 你主要在国内团队协作里用，就优先看飞书。
- 你主要是个人使用、海外使用，或者更看重 Bot 生态，就优先看 Telegram。
- 你做海外私域或客服，需要覆盖 WhatsApp 用户，就单独评估 WhatsApp。
- 你是社区、游戏或开源项目场景，就重点看 Discord。
- 如果你需要多个入口，可以同时接多个 Channel，让它们共用同一套 Agent 能力和记忆。

### 7. 本章避坑诊断（Troubleshooting）

#### 7.1 飞书相关问题

- **飞书机器人无响应**：首先检查 `openclaw.json` 中的飞书配置是否正确，确认 `appId` 和 `appSecret` 填写无误。然后检查 `connectionMode` 是否与你的网络环境匹配（本地开发推荐 `websocket`）。

- **飞书消息卡片渲染异常**：部分旧版本飞书客户端对消息卡片的兼容性不佳，可以尝试升级飞书客户端到最新版本。

- **权限不足导致功能受限**：对照 2.4 节的权限列表，在飞书开放平台的"权限管理"页面检查是否有遗漏的权限未开通。

#### 7.2 Telegram 相关问题

- **Bot 无法收到消息**：先确认 `botToken` 是否正确。Telegram 默认就走 Long polling，不需要额外把 `polling: true` 写成必填项；如果你主动切到了 Webhook 模式，再检查公网地址和 HTTPS 证书。

- **消息格式错乱**：Telegram 的 MarkdownV2 对特殊字符（如 `.`、`-`、`(`、`)`）要求转义。如果 AI 回复的消息经常格式错乱，可以检查 OpenClaw 的日志排查具体原因。

- **群聊中 Bot 不回复**：Telegram 的 Bot 默认无法读取群聊中的所有消息（Privacy Mode）。你需要在 BotFather 中通过 `/setprivacy` 命令将 Privacy Mode 设置为 `Disable`，或者确保每条消息都 @ 了 Bot。
