#### 2.1 能“做事”的 AI

大多数人第一次听到“AI 助手”，脑子里浮现的是 ChatGPT、Claude、Gemini：你问，它答，很聪明，但仅限于对话框里。

OpenClaw 完全不同。它从设计第一天起就不是聊天工具，而是 **行动代理（Action Agent）** 。

核心区别对比：

| 维度   | 传统 LLM 聊天（如 [Claude.ai ](http://Claude.ai)） | OpenClaw                                        |
| ---- | ------------------------------------------- | ----------------------------------------------- |
| 运行位置 | 云端                                          | 本地 / 自托管 VPS                                    |
| 数据隐私 | 全部上传给厂商                                     | 默认本地，零上传（除非你用云模型）                               |
| 能做什么 | 回答、写代码、分析文本                                 | 执行真实操作：发邮件、日历、浏览器填表、读写本地文件、运行终端命令               |
| 记忆方式 | 对话轮次上下文（几千 token）                           | 持久化长记忆（跨会话、跨设备、几个月都记得你喜欢喝什么咖啡）                  |
| 交互通道 | 只能网页/App                                    | WhatsApp、Telegram、Discord、Slack、iMessage、甚至语音唤醒 |
| 扩展方式 | Prompt 调教                                   | Skills 系统（社区市场 + 自己写代码）                         |
| 危险系数 | 低（最多胡说八道）                                   | 极高（能真的改你文件、发你邮件、转你钱）                            |

一句话总结： **ChatGPT 是你的外脑，OpenClaw 是你的数字分身** 。

#####  五大核心概念（必须记住）

1.  **Gateway** OpenClaw 的“大脑”。一个常驻的 Node.js 进程（默认端口 9234 WebSocket）。 它负责：

    1.  接收所有通道的消息
    1.  维护所有 Agent 会话
    1.  调用 LLM 做决策
    1.  执行工具调用（Tools → RPC）
    1.  把结果推回聊天界面

1.  **Agent** 每一个和你聊天的“人格”。 可以有多个 Agent：

    1.  “工作分身” —— 只管日历、邮件、Notion
    1.  “生活分身” —— 订外卖、查天气、记账
    1.  “代码分身” —— 专门帮你写/改代码 每个 Agent 有独立的：
    1.  记忆文件（JSON + embeddings）
    1.  自定义 Prompt（SOUL.md / AGENTS.md）
    1.  可用技能白名单

1.  **Skills** OpenClaw 的“超能力”来源。 三种类型：

    1.  **Bundled** ：官方自带（浏览器控制、文件读写、终端执行等）
    1.  **Managed** ：从 ClawHub 一键安装
    1.  **Workspace** ：你自己写的（放在 ~/.openclaw/workspace/skills/ 目录） 每个 Skill 包含：
    1.  SKILL.md（自然语言描述 + 工具定义）
    1.  TypeScript 工具函数（可选）
    1.  依赖的其他 Skill（链式调用）

1.  **Channels** OpenClaw 能“听”和“说”的地方。 已支持 20+：WhatsApp、Telegram、Discord、Slack、iMessage（BlueBubbles）、Signal、Matrix、Feishu、LINE、Zalo…… 未来还会继续长。

1.  **Nodes** “身体”部分。运行在手机、平板、Mac 菜单栏的轻量客户端。 提供本地能力：拍照、屏幕录制、麦克风输入、GPS、系统通知等。



#### 2.2 “龙虾哲学”到底是什么？

创始人 steipete 在 VISION.md 里反复提到： **The lobster way** 。

翻译成人话大概是：

-   先把“钳子”练得特别锋利（工具调用要稳、快、准）
-   再慢慢长壳（加安全边界、沙箱、审批流）
-   不怕难看，先能用（MVP 极致实用主义）
-   社区共生，像龙虾一样脱壳成长（ClawHub 公开、Skills 开源）

这也是为什么 OpenClaw 发布才几个月，却能做到很多商业产品几年都做不到的事：真正的 **自主执行 + 持久记忆 + 多通道** 。

#####  常见误区与风险提醒（第1章就要打预防针）

误区1：以为它和 Claude / GPT 一样安全

→ 完全不是。它有真实执行能力，相当于把 Claude 装进你电脑里，还给了它你的邮箱密码、日历密钥、文件读写权限。

误区2：随便从 ClawHub 安装技能 → 截至 2026 年 3 月，ClawHub 上线两个月，已经发现数百个恶意技能（窃取信息、加密货币骗局、后门）

误区3：把它当普通聊天机器人用

→ 那就浪费了 90% 的价值。OpenClaw 的杀手级场景是“无人值守自动化”。

小结：OpenClaw = 强大 × 危险 × 自由。

你拥有的控制权越多，它能帮你做的事就越多；反之亦然。

#####  前置环境配置

安装 OpenClaw 其实非常简单——官方把门槛降到了“一条命令 + 5 分钟 wizard”。但简单不等于没坑，尤其是你想让它 **稳定 24/7 运行** 、 **连接真实通道** （WhatsApp/Telegram 等）或 **开启高危工具** （浏览器控制、文件读写、shell 执行）时，提前准备好环境能避免 80% 的挫败感。


#####  系统要求（2026年3月最新）

| 项目      | 最低要求                            | 推荐配置（日常/生产）                      | 备注                    |
| ------- | ------------------------------- | -------------------------------- | --------------------- |
| 操作系统    | macOS 12+ / Windows 10+ / Linux | macOS 14+ / Ubuntu 22.04+ / WSL2 | Windows 原生支持较弱，推荐 WSL |
| Node.js | ≥22（官方强烈推荐 22.x LTS）            | 22.9+ 或最新 23.x                   | 低于 22 会报大量 ESM 错误     |
| 内存      | 4GB                             | 8–16GB+（多 Agent + 浏览器工具）         | 本地模型（如 Ollama）需 16GB+ |
| 磁盘      | 5GB 空闲                          | 20GB+（模型缓存 + 记忆文件）               | ~/.openclaw 会长大      |
| 网络      | 能访问 openai/anthropic/google 等   | 稳定宽带 + 科学上网（API 调用）              | 中国大陆用户需注意             |
| 权限      | 管理员/ sudo                       | 专用用户或 Docker                     | 高危工具需系统权限

**重要提醒** ： OpenClaw 默认会请求 **浏览器控制** 、 **文件读写** 、 **终端执行** 等权限。 **不要** 在主力电脑/公司电脑上第一次就开全部权限。建议：

-   第一台机器用虚拟机（Parallels/UTM/VirtualBox）
-   或买个二手 Mac mini / NUC / VPS 专门跑
-   至少先用 **只读工具** + **本地模型** 测试

#####  常见安装坑与解决（真实反馈 Top 10）

1.  **curl install.sh 超时/连接拒绝** → 用代理或手动下载脚本
1.  **Node 版本不对** → node -v 确认 ≥22，必要时 nvm use 22
1.  **Windows 权限问题** → PowerShell 以管理员运行，或用 WSL
1.  **Ollama 连接失败** → 确认 Ollama running & api 在 11434
1.  **Dashboard 打不开** → 检查端口 9234 是否被占用（lsof -i:9234）
1.  **API Key 无效** → 复制时不要带空格，检查配额
1.  **安装后 gateway 不启动** → openclaw gateway logs 查看错误
1.  **内存爆炸** → 关掉浏览器工具，或限制并行 Agent 数量
1.  **中国大陆 API 连不上** → 用 OpenRouter 或本地 Ollama
1.  **想卸载重来** → npm uninstall -g openclaw + rm -rf ~/.openclaw

小结：安装本身不难，难的是 **后续的安全与稳定** 。下一章我们直接上手第一个真实交互——从聊天框里让它帮你做事。

#####  第一次失败怎么办？（必看）

最常见的3种情况及解决：

1.  **工具调用被拒绝** → 因为你第2章开启了“高危操作需审批”。在弹出的确认框点“允许本次”或“始终允许该工具”。
1.  **浏览器没反应** → 第一次会要求你选择默认浏览器并授予“自动化控制”权限（macOS 系统偏好设置 → 隐私与安全性 → 自动化）。
1.  **回复很慢** → 输入 /think fast 切换模式，或临时换成更快模型（gpt-4o-mini / gemini-flash）。
