###  架构深度解析

很多人把 OpenClaw 当成“一个很聪明的聊天机器人”，其实大错特错。 它更像是一台 **微型分布式操作系统** ，专门为 AI 代理量身定制。

本章我们从“上帝视角”把整个系统拆开来看：Gateway 是大脑，Agent 是灵魂，Tools 是手脚，Channels 是耳朵嘴巴，Nodes 是身体。理解它们之间的数据流，你就真正“看懂”了 OpenClaw。

#### 3.1 整体架构图（建议截图保存）


![exported_image.png](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf8e2698a76444c88af211b67f040204~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1013&h=1840&s=120015&e=png&b=ffffff)

**核心设计哲学** ： **一切通过 WebSocket 双向流** 。没有 REST API，没有 HTTP 轮询，一切都是实时、流式、事件驱动。这也是为什么它能做到“秒级响应 + 实时看到浏览器在动”的根本原因。

#####  Gateway：整个系统的大脑（必须常驻）

-   **进程名称** ：openclaw-gateway（Node.js 22+）

-   **默认端口** ：9234（WebSocket + HTTP Dashboard）

-   **主要职责** （按重要性排序）：

    -   维护所有 WebSocket 连接（Channels）
    -   路由消息到对应 Agent
    -   调用 LLM 并解析 tool_calls
    -   执行 Tool RPC 并把结果流式返回
    -   管理内存、压缩、备份

**查看 Gateway 实时信息** （强烈推荐每天看一次）：

Bash

```
openclaw gateway status        # 简要状态
openclaw gateway logs --follow # 实时日志（按 Ctrl+C 退出）
openclaw gateway metrics       # CPU/内存/并发 Agent 数
```

**关键配置** （~/.openclaw/openclaw.json）：

JSON

```js
{
  "gateway": {
    "port": 9234,                // Gateway服务监听的端口（对应之前的WebSocket端口）
    "maxAgents": 20,             // 支持的最大Agent实例数
    "memoryLimitMB": 4096,       // Gateway进程的内存限制（单位：MB）
    "enableSandbox": true        // 是否启用沙箱模式（隔离Agent运行环境，提升安全性）
  }
}
```

**小贴士** ：生产环境建议把 Gateway 跑在单独的 VPS 或 Docker 里，用 Tailscale 暴露 9234 端口即可。

##### Agent：你的数字分身（独立人格）

每个 Agent 都是一个 **隔离的会话 + 记忆 + 工具白名单** 。

-   **存储位置** ：~/.openclaw/agents/**agent-id**/

    -   soul.md：核心人格提示词（你改这里它就变性格）
    -   memory.json + embeddings/：长期记忆
    -   tools.json：当前可用 Skill 白名单

**创建/切换 Agent** ：

-   Dashboard 左上角切换
-   命令：/new 
-   命令：/switch

**Agent 状态机** （内部原理）：

1.  Idle → 收到消息
1.  Planning（LLM 思考下一步）
1.  Tool Calling（并行或串行）
1.  Executing（真实执行）
1.  Responding（流式返回给用户）
1.  Committing Memory（保存到长期记忆）

#####  Tools 系统：真正的“执行力”来源

OpenClaw 所有能力都来自 **Tool** （工具调用）。 目前（2026年3月）内置 28 个官方 Tool，分 5 大类：

| 类别            | 代表 Tool                          | 危险等级  | 备注       |
| ------------- | -------------------------------- | ----- | -------- |
| Browser       | browser_navigate, click, type    | ★★★★★ | 最强也最危险   |
| Filesystem    | read_file, write_file, list_dir  | ★★★★  | 默认沙箱     |
| Shell         | terminal_exec                    | ★★★★★ | 需审批      |
| Communication | send_email, calendar_create      | ★★★★  | 需要 OAuth |
| Media         | transcribe_audio, vision_analyze | ★★    | 本地节点提供   |

##### Live Canvas（A2UI）：动态界面革命

这是 OpenClaw 2025年底才上线的杀手级特性。

当 Agent 需要复杂交互时，不再只发文字，而是 **实时推送一个动态 UI** 到 Dashboard（或 Channel 支持的富媒体）。

例子：

-   让它帮你规划旅行 → 它会直接弹出一个可交互的行程表格（可点“确认订票”）
-   代码审查 → 实时高亮 diff + 一键应用修改

技术上：Gateway 通过 WebSocket 推送 JSON UI Schema，浏览器用 React 渲染。

#####  完整数据流示例（一次浏览器搜索的内部过程）

1.  你在 WhatsApp 说：“帮我搜 OpenClaw 最新版本”
1.  Channel → Gateway（WebSocket）
1.  Gateway 路由到对应 Agent
1.  Agent 调用 LLM → 决定用 browser_navigate
1.  Tool 执行器在沙箱里打开 Chrome
1.  结果流式返回 → Agent 总结
1.  通过原 Channel 发回给你，同时在 Dashboard 也显示

整个过程平均 6-12 秒，无需你干预。

##### 安全边界在哪里？（架构级防护）

-   **每一次 Tool 调用** 都经过三层检查：

    -   Skill 白名单
    -   用户审批流（高危工具）
    -   WASM 沙箱（文件/网络隔离）

-   **提示注入防御** ：Gateway 有内置的 prompt guard（但不是 100% 有效）

-   **内存隔离** ：每个 Agent 独立进程池

**作者金句** ： “架构决定了上限，安全决定了是否敢用上限。”




####  3.2 配置全攻略

第2章的 onboarding wizard 已经帮你生成了一个最小可用的配置。但真正的威力，藏在 ~/.openclaw/openclaw.json 这个文件里。

**核心事实**（必须记住）：

-   格式：**JSON5**（支持注释、尾随逗号，比 JSON 好写多了）

-   位置：~/.openclaw/openclaw.json（或通过环境变量 OPENCLAW_CONFIG_PATH 自定义）

-   热重载：修改后 **Gateway 自动生效**，无需重启！

-   编辑方式推荐：

    -   openclaw config（CLI 向导，最安全）
    -   openclaw config get/set（精确修改）
    -   VS Code 直接打开文件（有语法高亮）

#####   查看当前配置（必做第一步）

Bash

```
# 1. 打印配置文件完整路径openclaw config file
# 2. 查看完整配置（带注释）openclaw config get
# 3. 查看特定路径（推荐）
openclaw config get channels.whatsapp
openclaw config get agents.defaults
openclaw config get browser
```

#####  最小推荐配置（复制即用）

把下面内容保存到 ~/.openclaw/openclaw.json（如果文件不存在就新建）：

```js
{
  // 代理默认设置
  "agents": {
    "defaults": {
      "workspace": "~/.openclaw/workspace", // 技能、记忆、提示词的根目录
      "maxMemoryTokens": 128000, // 最大记忆令牌数（适配Claude 3.7 128k上下文）
      "autoCompact": true // 自动压缩旧记忆，节省令牌
    }
  },
  // 通道安全配置（生产环境必配）
  "channels": {
    "whatsapp": {
      "allowFrom": ["+861xxxxxxxxxx"], // 白名单：仅允许指定手机号发起请求
      "groups": {
        "*": {
          "requireMention": true // 所有群聊需@openclaw才响应，避免刷屏
        }
      }
    },
    "telegram": {
      "botToken": "${TELEGRAM_BOT_TOKEN}" // 从环境变量读取BotToken，避免明文泄露
    }
  },
  // 浏览器控制配置（高危操作，需严格限制）
  "browser": {
    "executablePath": "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome", // macOS Chrome路径
    "headless": false, // 非无头模式：可直观看到浏览器操作过程
    "sandbox": true // 强制开启沙箱：防止恶意操作
  },
  // 全局安全策略
  "security": {
    "requireApproval": ["high", "medium"], // 中/高危操作需手动确认
    "sandboxEnabled": true, // 全局沙箱开关
    "logToolCalls": true // 记录所有工具调用日志，便于审计
  }
}
```



**保存后**，运行 openclaw gateway restart 或等待 3 秒自动热重载。

#####  完整配置参考表（2026年3月最新）

| 类别   | 关键路径                               | 推荐值 / 示例                          | 说明              |
| ---- | ---------------------------------- | --------------------------------- | --------------- |
| 代理默认 | agents.defaults.workspace          | ~/.openclaw/workspace             | 技能市场安装位置        |
| 代理默认 | agents.defaults.maxConcurrentTools | 5                                 | 防止并行工具爆炸        |
| LLM  | llm.provider                       | "anthropic" / "openai" / "ollama" | wizard 设置后自动生成  |
| LLM  | llm.model                          | "claude-3.7-sonnet"               | 可动态切换           |
| 浏览器  | browser.executablePath             | 具体路径                              | 必填否则工具失效        |
| 浏览器  | browser.userDataDir                | "~/.openclaw/browser-profile"     | 独立配置文件，不污染你主浏览器 |
| 通道   | channels.*.allowFrom               | 手机号/用户ID 数组                       | 白名单，必开！         |
| 通道   | channels.*.groups.*.requireMention | TRUE                              | 防群聊刷屏           |
| 工具安全 | tools.*.requireApproval            | ["terminal_exec", "write_file"]   | 高危工具审批          |
| 性能   | gateway.maxAgents                  | 10                                | 根据内存调整          |
| 性能   | gateway.memoryLimitMB              | 8192                              | 防止 OOM          |
| 日志   | logging.level                      | "debug" / "info" / "warn"         | 排查问题神器          |

完整参考见官方：<https://docs.openclaw.ai/gateway/configuration-reference>

#### 3.3 常用 CLI 配置命令（强烈推荐）

Bash

```
# 设置浏览器路径openclaw config set browser.executablePath "/usr/bin/google-chrome"
# 添加 WhatsApp 白名单openclaw config set channels.whatsapp.allowFrom '["+8612345678901"]' --strict-json
# 开启所有群聊必须@ openclaw config set channels.whatsapp.groups '{"*": {"requireMention": true}}'
# 使用环境变量（最高安全）openclaw config set channels.telegram.botToken '${TELEGRAM_BOT_TOKEN}'
# 验证配置（防语法错误）openclaw config validate
# 重置为安全默认值openclaw config reset --safe
```

#####  环境变量（生产必备）

在 ~/.openclaw/.env 或系统环境变量里设置：

Bash

```
ANTHROPIC_API_KEY=sk-...
TELEGRAM_BOT_TOKEN=123456:ABC...
OPENCLAW_CONFIG_PATH=/opt/openclaw/config.json
```

配置文件里用 ${VARIABLE_NAME} 引用，OpenClaw 会自动替换（绝不把密钥明文写进 json）。

#####  群聊 vs 私聊隔离策略（防刷屏神器）

默认配置下群聊容易被刷屏，第5章我们已经加了 requireMention: true。这里给出更极致的玩法：

在 openclaw.json 添加：

```js
{
  "channels": {
    "whatsapp": {
      "groups": {
        "家庭群": {
          "autoReply": false,
          "onlyWhenMentioned": true
        },
        "工作项目群": {
          "agent": "工作分身"
        }
      }
    }
  }
}
```

**实用技巧** ：

-   私聊：全权交给“生活分身”
-   工作群：只在被 @ 时回应，且必须先说“已阅，正在处理”
-   家庭群：只处理特定关键词（如“帮我订外卖”）

#####  持久记忆技巧（让它越来越懂你）

Agent 的记忆是 OpenClaw 最强的地方，但默认会慢慢膨胀。掌握以下 4 个命令，记忆永不失控：

| 命令              | 作用            | 推荐频率 |
| --------------- | ------------- | ---- |
| /compact        | 强制压缩记忆（推荐）    | 每天一次 |
| /remember 关键信息  | 手动写入长期记忆      | 重要事情 |
| /forget 项目名称    | 删除特定记忆        | 任务结束 |
| /memory summary | 查看当前记忆摘要（超实用） | 每周一次 |

**高手玩法** ： 在 ~/.openclaw/agents/主Agent/soul.md 最前面加一段：

text

```
你是一个极度注重隐私和效率的个人助理。记住以下用户偏好：- 咖啡永远不要加糖- 会议默认提前10分钟提醒- 所有文件自动按“年/月/主题”归档
```

保存后它会立刻“记住”并终身执行。

#####  每日例行自动化（真正解放双手）

每天早上 8:00 自动执行的经典指令（发给它一次，它会自己记住）：

从今天开始，每天早上 8:00 自动执行以下任务：1. 查看我的 Gmail 未读邮件，总结重要事项2. 看今天天气和日历，告诉我穿什么、几点出门3. 把昨天的支出记录到 Notion 表格回复“已设置每日例行”即可。
