###  通道与设备集成

**核心理念** ： Channel = OpenClaw 的“耳朵和嘴巴”。 一旦接入，你就可以在原本的聊天 App 里直接和你的 Agent 对话，它会像真人一样回复、执行任务、发文件、分享链接。

**安全第一原则** ：

-   永远只接入 **你自己的账号**
-   必须开启白名单（allowFrom）
-   群聊默认 **必须 @ 才回应**
-   高危工具（如 terminal_exec）在 Channel 里默认禁用审批流

#### 4.1  主流聊天应用接入（最推荐，80% 用户首选）

WhatsApp 是 OpenClaw 支持最完善、体验最好的通道。

**配置步骤（15 分钟）**

1.  **准备工作**

    1.  手机保持在线（或用 WhatsApp Web 多设备登录）
    1.  确保你的手机号已验证

1.  **命令行配置（推荐）**

Bash

```
openclaw channels add whatsapp
```

系统会弹出二维码：

text

```
[QR Code 出现]请用手机 WhatsApp → 设置 → 已链接设备 → 扫码登录
```

扫码后自动完成认证。

2.  **安全加固（必须做！）**

Bash

```
# 只允许你自己的手机号对话
openclaw config set channels.whatsapp.allowFrom '["+861xxxxxxxxxx"]'
# 所有群聊必须 @openclaw 才回应（防刷屏）
openclaw config set channels.whatsapp.groups '{"*": {"requireMention": true, "autoReply": false}}'
# 开启端到端加密确认
openclaw channel whatsapp --enable-e2e
```

2.  **验证**

在你的 WhatsApp 里给自己的号发消息：

text

```
@openclaw 你好，测试一下
```

它应该 3-8 秒内回复，并显示 🦞 图标。

**高级玩法** ：

-   支持群组白名单：只有指定群才响应
-   支持语音消息自动转文字 + 执行
-   支持发位置、图片、文档

####  4.2 设备节点（Nodes）与移动扩展

**核心概念** ： Gateway 是大脑，Channel 是耳朵嘴巴， **Node 就是身体** 。 一个 Node 就是一个轻量客户端（<30MB），跑在你的设备上，提供 **本地硬件能力** ，并通过 WebSocket（9234 端口）和 Gateway 实时通信。

**支持的 Node 类型** （2026年3月）：

-   macOS 菜单栏 App（最丝滑）
-   iOS App（TestFlight / App Store）
-   Android App（Google Play / APK）
-   Windows 托盘（实验性）
-   Linux 系统托盘（实验性）

**为什么 Node 这么重要？** 没有 Node，你只能控制浏览器和文件；有了 Node，你可以：

-   手机拍一张发票照片 → 自动记账 + 存云盘
-   Mac 菜单栏一键截屏 → AI 总结内容
-   开车时说“Hey Claw” → 手机节点自动导航 + 语音播报

#####  macOS 菜单栏 Node（推荐首装）

**安装步骤（3 分钟）**

Bash

```
# 通过官方一键安装（推荐）openclaw node install macos
# 或者从官网下载最新版 https://openclaw.ai/download/node-macos
```

安装后：

-   菜单栏出现 🦞 小龙虾图标

-   右键 → 快速指令菜单（截屏、录音、打开 Dashboard 等）

**授权必须项** （第一次会弹出）：

-   屏幕录制权限（系统偏好设置 → 隐私与安全性 → 屏幕与系统录制）
-   麦克风权限
-   相机权限（可选）

**测试** ： 点击菜单栏龙虾 → “快速截屏分析” 选中任意区域 → 它会自动把截图发给当前 Agent 分析并回复。

-   设置 → 隐私与安全性 → 相机 / 麦克风 → 始终

**杀手级功能** ：

-   “拍照即分析”：拍发票/菜单/名片 → 秒级结构化保存
-   “语音唤醒”：全局“Hey Claw”支持（需开启 Siri 快捷指令）

#####  多设备协同实战案例（最爽的部分）

```js
      },

          "battery_optimized"

  }

```

#####  安全与隐私提醒（Node 权限最高！）

Node 是唯一能直接访问你 **相机、GPS、屏幕** 的组件：

-   永远不要在公共 Wi-Fi 上暴露 Gateway 端口
-   建议用 `Tailscale`做零信任网络
-   高危节点操作（如屏幕录制）默认走审批流

**小技巧** ：在节点设置里可以单独给每个 Node 设置“能力白名单”，比如只给 iPhone 开相机，不开麦克风。

#### 4.3 远程访问与服务器部署

**为什么需要远程部署？**

-   本地电脑关机 → Agent 就死了
-   手机/出差时想用 → 必须能随时连上
-   24/7 自动化（每天早报、航班值机、监控邮箱）→ 必须永不掉线

**核心原则** ： **永远不要把 9234 端口直接暴露到公网！** 这是 OpenClaw 安全事故最多的原因（ClawJacked 漏洞就是这么来的）。

我们按 **从简单到硬核** 的顺序来，99% 用户选前两种就够了。

##### 最推荐：Tailscale 一键零信任访问（5 分钟搞定）

Tailscale 是 OpenClaw 官方最推的远程方案（零配置、端到端加密、全球加速）。

**步骤**

Bash

```
# 1. 在你的 VPS / 本地机器都安装Tailscale 
openclaw node install tailscale   
# 2. 一键登录并暴露 Gateway
openclaw gateway expose --tailscale
```

命令会自动：

-   把 Gateway 绑定到 Tailscale IP（不会暴露公网）
-   生成分享链接（发给手机/iPad 也能秒连）
-   开启 Funnel（可选，让特定 Channel 从外网安全访问）

**验证** ： 在手机上打开 Tailscale App → 看到你的机器 → 浏览器输入 http://你的机器名:9234 就能打开 Dashboard！

**优势** ：

-   完全免费（个人 3 台设备够用）
-   穿透防火墙、CGNAT
-   支持多设备（Mac + VPS + 手机同时在线）

#####  生产级：VPS + Docker 24/7 部署（推荐 Hostinger / DigitalOcean）

**推荐配置** （每月 5-8 美元就够）：

-   2 vCPU + 4-8GB RAM（Hostinger VPS 最划算）
-   Ubuntu 22.04 LTS

**完整部署命令（复制粘贴）**

Bash

```
# 1. 更新系统sudo apt update && sudo apt upgrade -y
# 2. 安装 Docker + Composesudo apt install docker.io docker-compose -ysudo usermod -aG docker $USER
# 3. 拉取官方镜像mkdir ~/openclaw && cd ~/openclawcurl -O https://openclaw.ai/docker-compose.yml
# 4. 编辑 docker-compose.yml（填你的 API Key）nano docker-compose.yml# 把 ANTHROPIC_API_KEY 等填好
# 5. 启动docker-compose up -d
```

**持久化** （重启不丢数据）：

YAML

```
volumes:  - ./openclaw-data:/root/.openclaw
```

**验证** ：

Bash

```
openclaw gateway status   # 应该显示 Runningdocker logs 
openclaw-gateway --follow
```

##### 远程部署安全 checklist（必须全部勾上！）

-   Gateway 只监听 localhost 或 Tailscale IP
-   所有 Channel 开启白名单 + requireMention
-   高危工具（terminal_exec、write_file）开启审批流
-   使用独立浏览器 profile（--user-data-dir）
-   定期 openclaw config validate
-   备份 ~/.openclaw 目录（每周 rsync 到另一台机器）
-   不要在 VPS 上装任何第三方 Skill，除非你亲自 review 代码

**真实案例** ：2026 年 2 月有用户把 VPS 9234 端口直连公网，被恶意 Skill 偷走所有浏览器 Cookie，损失 3 个比特币。 **教训惨痛** 。
