# 第 9 章：实战二：本地文件管理与日程自动化

> 上一章，我们用三个 AI Agent 搭建了一条全自动的内容生产流水线。但回到日常工作中，真正吞噬你时间和精力的，往往不是那些"大项目"，而是无穷无尽的琐碎杂务：满硬盘翻找文件、肉眼核对发票贴报销单、把微信群里的开会截图手动录入日历、从长篇会议录音里提炼待办事项……这些事情单独拿出来都不难，但加在一起，每天可能占用你不少有效工作时间。

本章我们将利用 OpenClaw 的本地系统集成能力，带你构建一套办公自动化方案。从文件管理、报销自动化，到日历同步、会议纪要生成，再到邮件摘要和小型客服——每一个场景都配有操作指令和配置思路供参考。需要注意的是，本章涉及的部分技能（skill）来自 ClawHub 社区或第三方，并非全部为官方内置能力，请以各技能页面的实际说明为准。

**本章每个场景是否能跑通，都取决于你安装了什么 skill，以及这些 skill 当前是否仍在维护。文中的命令和配置主要用于说明实现思路，请始终以 ClawHub 页面和对应 skill 文档为准。**

## 本章学习目标

读完本章，你将能够：

*   了解 OpenClaw 的图像识别能力，实现发票的智能搜索与信息提取

## 1. 场景一：本地文件管理神器与报销单自动化

> 这一节涉及 OCR、文件读写、表格处理等能力。是否可用，取决于你安装了什么 skill，以及对应 skill 当前是否仍在维护。

普通 AI 只能在聊天窗口里和你对话，而部署在本地的 OpenClaw 可以直接读取、操作你电脑上的文件。这意味着它可以帮你处理一些原本需要手动完成的文件操作任务。

### 1.1 前置准备：确认图像识别能力

要让 AI 能"看懂"发票图片，需要具备图像识别（OCR）相关的能力。OpenClaw 本身的多模态能力已经可以理解图片内容；如果你需要更专业的 OCR 或文件管理技能，可以在 ClawHub 上搜索对应的技能包，以技能页面的安装说明为准。

```bash
# 在 ClawHub 搜索图像识别相关技能
clawhub search vision ocr

# 安装你找到的技能（以实际 slug 为准）
clawhub install <skill-slug>

# 查看当前可见的技能列表
openclaw skills list
```

> 注意：以下示例中涉及的 OCR 识别、文件管理、表格处理等能力，取决于你实际安装的技能或 OpenClaw 的内置多模态能力。不同技能的命令和参数可能有所不同，请以技能页面说明为准。

安装完成后，在 OpenClaw 的对话界面中输入以下指令确认技能已生效：

    你好，请列出你当前已安装的所有技能，特别是和图像识别、文件管理相关的。

### 1.2 智能找发票：语义搜索 + 图像识别

传统的文件搜索只能按文件名匹配，但你的发票截图可能叫 `IMG_20260301_142356.jpg` 这种毫无意义的名字。OpenClaw 的图像识别技能可以"看懂"图片内容，实现真正的语义搜索。

在对话框中输入以下指令：

    帮我从 ~/Downloads 目录中的所有图片里，找到上个月（2026年2月）购买跑步机的那张发票。
    搜索范围包括 jpg、png、heic 格式的文件。


AI 的处理流程如下：

1.  扫描 `~/Downloads` 目录下所有符合格式的图片文件
2.  对每张图片调用 OCR 识别，提取文字内容
3.  用语义匹配找到包含"跑步机"相关关键词的发票
4.  返回匹配结果，包括文件路径和识别出的关键信息

你还可以用更灵活的方式搜索：

    帮我找到最近三个月内所有金额超过 500 元的发票图片，按金额从高到低排列。

<!---->

    在 ~/Documents/发票 文件夹中，找到所有餐饮类的发票，并统计总金额。

### 1.3 批量填写报销单：从识别到填表一气呵成

找到发票只是第一步，真正耗时的是把发票信息一条条手动填进 Excel 报销单。让 AI 来接管这个苦差事。

首先，准备好你公司的报销单模板。假设模板文件位于 `~/Documents/报销单模板.xlsx`，包含以下列：日期、发票号码、消费类型、商品明细、金额、备注。

然后在对话框中输入：

    请帮我完成以下报销单填写任务：

    1. 扫描 ~/Desktop/本月发票/ 文件夹中的所有发票图片
    2. 识别每张发票上的：日期、发票号码、消费类型、商品明细、金额
    3. 将识别结果按日期排序，填入 ~/Documents/报销单模板.xlsx 中
    4. 如果有识别不确定的字段，请标记为黄色高亮并在备注列说明
    5. 完成后告诉我总金额和发票数量

### 1.4 硬盘大扫除：智能分类、归档与清理

电脑用久了，桌面和下载文件夹就会变成一个"数字垃圾场"。让 AI 帮你做一次彻底的大扫除。

#### 基础清理：按规则整理文件

    请帮我整理 ~/Downloads 文件夹，按以下规则操作：

    1. 将所有 .pdf 文件移动到 ~/Documents/PDF归档/
    2. 将所有图片文件（jpg/png/heic/webp）移动到 ~/Pictures/下载图片/
    3. 将所有 .dmg 和 .pkg 安装包移动到 ~/Downloads/安装包/
    4. 将超过 6 个月未访问的文件移动到 ~/Archive/待清理/
    5. 删除所有 .DS_Store 和 Thumbs.db 系统缓存文件
    6. 操作前先列出计划，等我确认后再执行

这里有一个关键细节：**最后一条"等我确认后再执行"非常重要**。它确保 AI 不会在你没看清楚的情况下直接移动或删除文件。AI 会先生成一份详细的操作计划：

    文件整理计划如下：

    📁 PDF 归档（17 个文件，共 234 MB）
      - 合同_2026Q1.pdf → ~/Documents/PDF归档/
      - 产品需求文档_v3.pdf → ~/Documents/PDF归档/
      ...

    🖼️ 图片归档（156 个文件，共 1.2 GB）
      - IMG_20260215_*.jpg (43个) → ~/Pictures/下载图片/
      ...

    📦 安装包归档（8 个文件，共 3.7 GB）
      - Docker.dmg → ~/Downloads/安装包/
      ...

    🗄️ 过期文件归档（23 个文件，共 890 MB）
      ...

    🗑️ 缓存清理（34 个文件）
      - .DS_Store × 34

    总计释放空间：约 6.0 GB
    是否确认执行？

#### 进阶清理：智能内容分类

如果你的文件命名混乱（比如全是 `未命名文档(1).docx`），可以让 AI 根据文件内容进行智能分类：

    请扫描 ~/Desktop 上的所有文档文件，根据文件内容自动分类：
    - 工作相关 → ~/Documents/工作/
    - 个人文档 → ~/Documents/个人/
    - 学习资料 → ~/Documents/学习/
    对于无法判断的文件，保留在原位并列出清单让我手动处理。
    同时，请将文件重命名为有意义的名称（保留原始扩展名）。

#### 定期自动清理：配置 Cron Job

如果你希望 AI 每周自动帮你整理一次，当前更推荐使用 `openclaw cron` 命令来管理任务，而不是直接在 `openclaw.json` 里手写 `cron` 数组：

```bash
openclaw cron add \
  --schedule "0 22 * * 5" \
  --agent main \
  --message "请执行每周文件整理任务：1) 整理 ~/Downloads 文件夹 2) 清理超过30天的缓存文件 3) 生成本周文件变动报告发送到飞书"
```

这条命令表示每周五晚上 10 点自动触发清理任务。AI 会在后台完成整理，并通过你已经接入的渠道发送清理报告。

## 2. 场景二："零手动"的日历与会议待办管家

> 这一节的日历同步、Todoist 联动等能力，也取决于你安装了什么 skill，以及这些 skill 当前是否仍在维护。

如果说文件管理解决的是"找东西"的问题，那么日历和会议管理解决的就是"不遗漏"的问题。老板在微信群里发了一张带有开会时间地点的截图，你需要来回切屏手动把信息添加到日历中——这种操作不仅繁琐，而且极其容易遗漏。

### 2.1 前置准备：配置日历集成

OpenClaw 支持与主流日历服务的深度集成。根据你使用的日历平台，选择对应的配置方案。

#### 方案 A：Apple Calendar 集成（macOS 用户推荐）

Apple Calendar 的集成可以利用 macOS 原生的 AppleScript 接口。如果你在 ClawHub 找到了对应的 Apple Calendar 技能，请以技能页面说明为准。以下是一般性的配置思路：

1.  在"系统设置 → 隐私与安全性 → 自动化"中，允许 OpenClaw（或终端应用）访问"日历"
2.  安装并配置对应的日历技能：

```bash
# 在 ClawHub 搜索 Apple Calendar 相关技能
clawhub search apple calendar

# 安装找到的技能（以实际 slug 为准）
clawhub install <skill-slug>
```

配置完成后，验证集成是否生效：

    请读取我 Apple 日历中今天的所有日程安排。

#### 方案 B：Google Calendar 集成（跨平台推荐）

Google Calendar 集成通常需要通过 OAuth 2.0 授权。如果你在 ClawHub 找到了对应的 Google Calendar 技能，请以技能页面说明为准。以下是一般性的配置思路：

1.  前往 [Google Cloud Console](https://console.cloud.google.com/) 创建一个项目
2.  启用 Google Calendar API
3.  创建 OAuth 2.0 客户端凭据，下载 `credentials.json`
4.  按照技能页面的说明完成授权配置

```bash
# 在 ClawHub 搜索 Google Calendar 相关技能
clawhub search google calendar

# 安装找到的技能（以实际 slug 为准）
clawhub install <skill-slug>
```

授权成功后，终端会显示已关联的账号和可访问的日历列表。

### 2.2 微信截图秒变日历事件

这是整个日历管家中最实用的功能。当老板或客户在微信群里发了一张包含会议信息的截图时，你只需要把截图转发给 OpenClaw。

如果你已经接入了飞书（参考第 4 章），直接在飞书中把截图发给 AI 即可。如果使用的是 Web 控制台，可以直接拖拽图片到对话框，并附上指令：

    请识别这张截图中的会议信息，并在我的日历中创建对应的日程提醒。
    如果有多个时间点，请分别创建。提前 15 分钟提醒我。

AI 的处理流程：

1.  调用图像识别能力识别截图中的文字内容
2.  用自然语言理解提取时间、地点、参与人、会议主题等结构化信息
3.  调用日历 API 创建事件，设置提醒
4.  返回确认信息，包含创建的事件详情

AI 返回的确认信息示例：

    已从截图中识别并创建以下日历事件：

    📅 事件：Q2 产品规划评审会
    📍 地点：公司 3 楼大会议室
    🕐 时间：2026-03-18（周三）14:00 - 16:00
    👥 参与人：张总、李经理、产品组全员
    ⏰ 提醒：提前 15 分钟

    已同步至 Apple Calendar → 工作日历

### 2.3 会议纪要自动生成与 Todoist 联动

开完会后，最头疼的就是整理会议纪要和提取待办事项。把会议的录音转写文本（或者你自己的简要笔记）发给 AI，它能帮你一步到位。

#### 会议纪要提示词模板

为了让 AI 每次都能输出格式统一、质量稳定的会议纪要，可以在工作区的 `AGENTS.md` 中配置一个专用的提示词模板。这是一种常见的写法，但不是官方唯一的配置方式——你也可以通过其他方式（如直接在对话中给出模板、使用 system prompt 等）来实现类似效果：

```markdown
## 会议纪要助手

当用户要求生成会议纪要时，请严格按照以下模板输出：

### 会议基本信息
- 会议主题：[从内容中提取]
- 会议时间：[从内容中提取]
- 参与人员：[从内容中提取]
- 会议时长：[从内容中提取或估算]

### 核心议题与结论
[按讨论顺序，列出每个议题的要点和最终结论]

1. **议题一：[标题]**
   - 讨论要点：...
   - 最终结论：...

### 决策事项
[列出本次会议做出的所有明确决策]

| 序号 | 决策内容 | 决策人 | 生效时间 |
|------|---------|--------|---------|

### 待办事项（Action Items）
[提取所有需要跟进的任务，明确责任人和截止时间]

| 序号 | 任务描述 | 责任人 | 截止时间 | 优先级 |
|------|---------|--------|---------|--------|

### 下次会议安排
- 时间：[如有提及]
- 议题预告：[如有提及]
```

使用时，只需将会议录音的转写文本发给 AI：

    以下是今天下午产品评审会的录音转写文本，请按照会议纪要模板生成纪要，
    并将其中属于我（张三）的待办事项同步到 Todoist。

    [粘贴转写文本]

#### 配置 Todoist 联动（第三方技能）

要实现待办事项自动同步到 Todoist，需要先在 ClawHub 查找并安装对应的 Todoist 集成技能。以下是一般性的配置思路，具体参数请以技能页面说明为准：

```bash
# 在 ClawHub 搜索 Todoist 相关技能
clawhub search todoist

# 安装找到的技能（以实际 slug 为准）
clawhub install <skill-slug>

# 按照技能说明配置 API Token（从 Todoist 设置 → 集成 → API Token 获取）
```

> 安全提示：API Token 属于敏感信息，建议使用环境变量或加密存储，不要明文写在配置文件里。

配置完成后，AI 在生成会议纪要时会自动将待办事项同步到 Todoist：

    会议纪要已生成并保存至：~/Documents/会议纪要/2026-03-18_产品评审会.md

    已同步以下待办事项到 Todoist：
    ✅ "完成竞品分析报告" → 项目：工作 | 截止：3月22日 | 优先级：P1
    ✅ "更新产品路线图 v2" → 项目：工作 | 截止：3月25日 | 优先级：P2
    ✅ "协调设计团队确认 UI 方案" → 项目：工作 | 截止：3月20日 | 优先级：P1

## 3. 场景三：收件箱整理与小型 AI 客服

> 这一节的邮件读取、知识库问答和客服接入，同样取决于你安装了什么 skill，以及这些 skill 当前是否仍在维护。

前两个场景解决了"文件"和"日程"两个方面的需求。接下来我们看看如何让 AI 帮你处理每天的邮件，以及搭建一个简单的客服应答系统。

### 3.1 每日邮件摘要：减少收件箱处理时间

如果你每天收到大量的新闻订阅、系统通知和工作邮件，逐封阅读既不现实也不高效。让 AI 每天早上帮你生成一份邮件摘要，可以帮助你快速筛选重点。

#### 配置邮件集成（第三方技能）

邮件集成需要在 ClawHub 查找对应的邮件读取技能。以下是一般性的配置思路，具体参数请以技能页面说明为准：

```bash
# 在 ClawHub 搜索邮件相关技能
clawhub search email reader

# 安装找到的技能（以实际 slug 为准）
clawhub install <skill-slug>

# 按照技能说明配置 IMAP 连接参数
```

> 安全提示：如果使用 Gmail，建议使用"应用专用密码"（App Password），
> 而非你的 Google 账号主密码。在 Google 账号设置 → 安全性 → 两步验证 → 应用专用密码 中生成。
> 密码等敏感信息建议使用环境变量存储，不要明文写在配置文件里。

#### 配置每日自动摘要

当前更推荐用 `openclaw cron add` 创建定时任务，让 AI 每天早上 8 点自动生成邮件摘要：

```bash
openclaw cron add \
  --schedule "0 8 * * *" \
  --agent main \
  --message "请读取我收件箱中过去 24 小时的所有未读邮件，按以下规则生成摘要：1) 将邮件分为【需要回复】【仅需了解】【可忽略】三个等级 2) 对【需要回复】的邮件，提取关键信息并建议回复要点 3) 对【仅需了解】的邮件，用一句话概括内容 4) 将摘要通过飞书发送给我"
```

AI 生成的每日邮件摘要示例：

    早安！以下是过去 24 小时的邮件摘要（共 47 封未读）：

    🔴 需要回复（3 封）
    1. 【客户-王总】关于 Q2 合作方案的确认
       关键点：对方希望在 3 月 20 日前收到修改后的报价单
       建议回复：确认收到，承诺周三前发送修改版

    2. 【HR-李姐】年度绩效面谈时间确认
       关键点：需要在本周五前选择一个面谈时间段
       建议回复：选择周四下午 3:00-4:00

    3. 【技术团队】生产环境部署审批
       关键点：v2.3.1 版本等待你的审批，包含 3 个 bug 修复
       建议回复：审批通过，建议在非高峰时段部署

    🟡 仅需了解（8 封）
    - GitHub：3 个 PR 已合并，1 个新 Issue
    - 产品周报：本周 DAU 环比增长 12%
    - 行业快讯：某竞品发布新功能（AI 辅助客服）
    - ...（其余 5 封为系统通知）

    ⚪ 可忽略（36 封）
    - 营销推广邮件 × 18
    - 系统自动通知 × 12
    - 新闻订阅 × 6

### 3.2 小型 AI 客服：自动应答方案

如果你经营个人品牌、小型电商或自由职业，客户咨询可能占用你不少时间。OpenClaw 可以帮你搭建一个简单的 AI 客服系统来处理常见问题。

#### 第一步：准备知识库文档

将你的业务信息整理成结构化文档，放在 OpenClaw 工作区中。你可以使用 OpenClaw 的 memory 或 workspace 目录来存放知识库文件，具体路径取决于你的 OpenClaw 版本和配置，请参考官方文档中关于 memory/workspace 的说明。以下是一个参考结构：

    <你的工作区目录>/knowledge/
    ├── 产品介绍.md          # 产品/服务的详细说明
    ├── 价格表.md            # 最新的价格和套餐信息
    ├── 退换货政策.md        # 售后政策和流程
    ├── 常见问题FAQ.md       # 客户高频问题及标准答案
    └── 话术指南.md          # 客服沟通的语气和边界规则

> 注意：`~/.openclaw/knowledge/` 并非官方标准路径。请以你当前 OpenClaw 版本的官方文档为准，使用正确的 memory 或 workspace 路径来存放知识库文件。

#### 第二步：配置客服 Agent

在工作区中创建一个专用的客服角色配置。以下示例使用 `AGENTS.md`，这是案例工作区的一种写法，不是官方唯一配置方式：

```markdown
## 客服助理（Customer Service Agent）

你是 [你的品牌名] 的在线客服助理。请严格遵守以下规则：

### 身份与语气
- 始终以品牌客服的身份回复，语气友好、专业、耐心
- 使用"您"作为称呼，避免过于随意的网络用语
- 每次回复控制在 3-5 句话以内，简洁明了

### 知识边界
- 只回答与产品、价格、售后相关的问题
- 所有回答必须基于 knowledge/ 目录中的文档，不得编造信息
- 如果文档中没有相关信息，回复："这个问题我需要转交给专属顾问为您解答，请稍等。"

### 转人工规则
遇到以下情况必须转交人工处理：
- 客户明确要求与人工客服对话
- 涉及投诉、纠纷或情绪激动的对话
- 涉及金额超过 5000 元的订单变更
- 连续 3 轮无法解决客户问题

### 转人工操作
转人工时，自动生成一份对话摘要，包含：客户问题、已尝试的解决方案、建议的处理方式，
通过飞书发送给 [你的飞书账号]。
```

#### 第三步：接入客户沟通渠道

根据你的业务场景，选择合适的接入方式：

**接入 WhatsApp Business（官方 channel 方式）：**

WhatsApp 当前官方支持通过 channel 登录流程接入：

```bash
# 使用官方 channel 登录流程
openclaw channels login --channel whatsapp
```

登录成功后，OpenClaw 即可通过 WhatsApp 接收和回复客户消息。具体配置细节请参考官方文档中关于 channels 的说明。

**接入企业微信：**

企业微信的接入目前没有官方内置支持。如果你在 ClawHub 找到了对应的企业微信技能，请以技能页面说明为准进行安装和配置。你也可以通过 Webhook 等方式自行对接企业微信的 API。

#### 第四步：测试与上线

配置完成后，先进行内部测试：

    请切换到客服助理角色，我来模拟客户提问进行测试。

    客户问：你们的退货政策是什么？超过 7 天还能退吗？

AI 会基于你的知识库文档回复：

    您好！感谢您的咨询。

    关于退货政策：自您收到商品之日起 15 天内，如商品未拆封且不影响二次销售，
    我们支持无理由退货。超过 7 天但在 15 天内的退货，运费需由您承担。

    如果商品存在质量问题，30 天内均可申请退换，运费由我们承担。

    请问您是遇到了什么问题需要退货吗？我可以帮您查看具体的处理方案。

### 4.2 实践建议

在实际使用中，有几个经验值得分享：

1.  **先小范围测试，再批量执行**：无论是文件整理还是报销单填写，第一次使用时建议先用少量文件测试，确认 AI 的理解和操作符合预期后，再放开批量处理。

2.  **善用"确认后再执行"机制**：在涉及文件移动、删除等不可逆操作时，务必在指令中加上"操作前先列出计划，等我确认后再执行"。这是你的安全网。

3.  **建立标准化的提示词模板**：对于高频使用的场景（如会议纪要、邮件摘要），花 10 分钟写一个好的提示词模板，能让后续每次使用都省下大量调整时间。将模板写入 `AGENTS.md` 是案例工作区的一种常见做法，但不是官方唯一配置方式，你也可以根据实际情况选择其他方式管理模板。

4.  **定期检查自动化任务的执行结果**：Cron Job 虽然方便，但不要完全"甩手不管"。建议每周花 5 分钟检查一下自动化任务的执行日志，确保没有异常。

5.  **注意隐私与数据安全**：邮件集成和日历集成涉及敏感的个人数据。确保你的 OpenClaw 实例运行在安全的环境中（参考第 10 章的安全红线），API Token 和密码使用环境变量或加密存储，绝不明文写在配置文件里。
